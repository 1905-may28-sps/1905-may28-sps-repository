About Data Binding

Angular supports data binding, which is a mechanism for coordinating parts of a template(HTML) with parts of a component (TS). We no longer use DOM element selection with angular, we simply add binding markup to our template HTML which will tell angular how to connect both sides

Interpolation

Interpolation is a way to display data from components in our templates, we can also use the same syntax for template expressions A few examples:
  • 2 + 2 = {{ 2 + 2 }}
  • text from component: {{ text | uppercase }}
  • Count = {{ count }}
  • More math: {{19849823/9}}
  • Using Math.random in TS file: {{someNum}}
  • Expressions : {{ count > 0 ? 'count is greater than 0' : 'no its not'}}
  • Current Date and time: {{currentTime | date: 'medium'}}
  • Currency w pipe: {{ 19379823748 | currency }}
  • Object with JSON pipe:
     {{ obj | json }}
Some rules about template expressions, they cannot be used with the following:
  • assignment operators (=, +=, -=, etc)
  • operators such as new, typeof, instanceof
  • chaining expressions with ; or ,
  • increment (++) or decrement (--)
  • no support for | &

Event Binding

The process of linking actions (methods) to events on DOM elements
  • This is one-way data binding
  • Syntax for event binding is to include 
    (event) = "function()"
    in your opening element tag

Count: {{ count }}

Property Binding

This one-way data binding mechanism allows us to apply a property to a DOM element dynamically. An inpurtant note about property binding is that it is frequently used with attribute directives, but it does not need to be.
  • Property binding syntax is as follows: 
    [property] = "value"

Two-Way Data Binding

This binding is likely the most powerful data binding technique as it allows us to take input from the user and immediately grab the data from the input field to be used in our template {{ myName }}
